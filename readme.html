READ ME

This project is 100% static – only HTML files with inline CSS/JS. Perfect for Nginx/Apache/S3/Netlify or a tiny Docker container.

Folder layout

/usr/share/nginx/html/
  index.html
  status.html
  logs.html
  downloads.html
  admin.html
  readme.html
  /downloads/
     tool-linux-amd64.tar.gz
     tool-windows-x64.zip
     tool-darwin-arm64.tar.gz

Run with Docker (Nginx)

docker run -d --name web \
  -p 80:80 \
  -v $PWD:/usr/share/nginx/html:ro \
  nginx:alpine

NGINX on Linux (no Docker)

sudo apt-get update && sudo apt-get install -y nginx
sudo cp *.html /usr/share/nginx/html/
sudo mkdir -p /usr/share/nginx/html/downloads
# put your artifacts into /usr/share/nginx/html/downloads/
sudo systemctl enable --now nginx

Apache

sudo apt-get install -y apache2
sudo cp *.html /var/www/html/
sudo mkdir -p /var/www/html/downloads
sudo systemctl enable --now apache2

Make health endpoint

Create a minimal health.html file (optional):

echo "<!doctype html><title>OK</title>OK" | sudo tee /usr/share/nginx/html/health.html

Wire to real data later

  • Replace mock metrics in status.html with fetch('/api/metrics').
  • Replace inline logs in logs.html with fetch('/logs') (server should stream or paginate).
  • Point downloads.html hrefs at your real artifacts paths.
  • Secure these pages behind auth at the reverse-proxy layer (e.g., Basic Auth in Nginx).

Quick Basic Auth (Nginx)

sudo apk add apache2-utils  # or: sudo apt-get install apache2-utils
htpasswd -bc auth admin S3cretP@ss
# nginx server block:
location / { auth_basic "Restricted"; auth_basic_user_file /etc/nginx/auth; }

All set. Open index.html to start.
